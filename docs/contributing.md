# Contributing

## Workflow

1. clone the repository
2. create a branch
3. make your change
4. run tests and formatting
5. open a pull request

## Before you submit

- update the docs when the public API changes
- keep examples aligned with the README
- make sure new configuration fields are documented

## Style

The codebase uses Black and isort, with formatting enforced through pre-commit.
